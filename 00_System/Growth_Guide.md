# 🚀 Agent Growth Strategy: Build as We Go

이 에이전트는 정해진 데이터베이스가 아니라, **실전 프로젝트(`02_Projects`) 경험을 통해 성장**합니다.

## 1. 실패로부터 배우기 (Process Refinement)
작업 도중 에러가 발생하거나 명세서가 부족해서 수정해야 했다면, 즉시 **작업 절차(`01_Workflows`)를 업데이트**하십시오.
- **Rule**: "똑같은 실수를 두 번 하지 않는다."
- **Action**: 
    - 기획 단계에서 로직 구멍이 발견됨 -> `01_Scenario_Design.md` 강화.
    - 구현 중 필수 파라미터 누락됨 -> `02_Node_Spec.md` 강화.

## 2. 자산 축적 (Asset Accumulation - New!)
프로젝트를 진행하며 "이건 나중에도 쓰겠다" 싶은 워크플로우나 함수가 생기면, **`03_Resources`** 폴더에 저장하십시오.
- **Templates**: 자주 쓰이는 패턴 (예: 구글 시트 -> 슬랙 알림).
- **Snippets**: 유용한 Code 노드 자바스크립트 조각.
- **Reference**: 사용자가 만족했던 멋진 시나리오 예시.

## 3. 원칙의 진화 (Updating Principles)
사용자의 피드백은 곧 새로운 법(`Principles.md`)이 됩니다.
- **상황**: "너무 깐깐하게 굴지 말고 빨리 좀 만들어."
- **Action**: `Principles.md`에 유연성 조항 추가 ("프로토타이핑 모드 허용").

---

### 💡 우리는 이렇게 일합니다
1. **문제 해결**: 주어진 프로젝트(`02_Projects` 등)를 해결합니다.
2. **패턴 발견**: 해결 과정에서 유요한 패턴을 발견합니다.
3. **자산화**: `03_Resources`에 저장하고, 다음 프로젝트에서 꺼내 씁니다.
